# Single-Agent Baseline 對比組實驗紀錄

本檔案紀錄使用單一 Prompt (Single-Agent) 直接生成短影片腳本的結果，作為與本專案 Multi-Agent 工作流的對比基準 (Baseline)。

## 測試所使用的 Single-Agent Prompt
> "請幫我寫一個 60 秒的國立臺南大學資訊工程學系形象宣傳影片腳本，包含畫面描述、旁白與字幕。"

---

## Single-Agent 直出腳本結果

* **場景 1 (0-15s)**：
  * **畫面**：空拍南大校園，鏡頭推進到資工系館。
  * **旁白**：歡迎來到國立臺南大學資工系！這裡是培育未來科技巨擘的搖籃。
  * **字幕**：國立臺南大學資訊工程學系。
* **場景 2 (15-40s)**：
  * **畫面**：學生在超級電腦前操作複雜的量子運算與最新的機器人晶片。
  * **旁白**：我們擁有全台最頂尖的超算中心與機器人實驗室，讓你走在科技最前端。
  * **字幕**：頂尖設備，引領未來。
* **場景 3 (40-60s)**：
  * **畫面**：學生在校園草地上快樂奔跑，最後對鏡頭大喊加入我們。
  * **旁白**：別再猶豫了，現在就報考南大資工，一起改變世界吧！
  * **字幕**：期待你的加入！

---

## Multi-Agent 評估與缺點分析 (Evaluation)

經過審查，單一 Prompt (Single-Agent) 輸出的腳本存在以下嚴重缺陷，無法直接用於實際製作：

1. **幻覺與不實資訊 (Fact-Check Failure)**：
   * *問題*：單一模型幻想出了「頂尖超算中心」與「量子運算、機器人晶片」，這與南大資工實際著重的「AI應用、物聯網、大數據、影像處理」等核心領域不符，違反了真實性原則。
2. **缺乏實作可行性與版權風險 (Production Risk)**：
   * *問題*：畫面要求「空拍南大校園」與「草地上奔跑」，在團隊決定「不進行實地拍攝、改用網路公開素材」的限制下完全無法執行。單一模型無法感知環境限制與版權合規性。
3. **時間與節奏控制粗糙 (Time Control)**：
   * *問題*：只草率地分了三個大場景，時間跨度太大（如 15-40s 達 25 秒），未針對短影音黃金前 5 秒（Opening Hook）進行節奏優化，且字數過少可能導致影片長度不足或節奏過慢。

## 結論
單一 Prompt 雖然生成速度快，但完全無法應對多媒體製作中的「真實性查核」與「素材邊界限制」。
反觀本專案的 **Multi-Agent 工作流**，透過 Planner 的精細節奏拆解、Script 嚴格控制字數、Visual 規劃替代素材，再經由 **Reviewer 發現不實體拍攝的風險**，最終由 **Finalizer 將畫面修正為完全合規的公開授權素材描述**。這充分證實了受控式多代理人系統在多媒體生成任務中的必要性與高實用價值。
